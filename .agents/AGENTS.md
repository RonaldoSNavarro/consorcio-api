# REGRAS DO PROJETO — CONSÓRCIO API

Ao trabalhar em qualquer tarefa do repositório `consorcio-api`, os agentes DEVEM:

1. **Iterações em Lotes Menores:** Ao elaborar planos de implementação (`implementation_plan.md`) ou executar listas de tarefas extensas, divida o trabalho em lotes pequenos e iterativos (Lote 1, Lote 2, etc.) e solicite aprovação/revisão do usuário entre os lotes.
2. **Nomenclatura Estrita (Domain-Driven):** O vocabulário de domínio do agente deve refletir fielmente a nomenclatura das entidades e enums do código-fonte (ex: `StatusCota`, `StatusLance`, `TipoMovimentoFinanceiro`). Evite generalizações do mercado que não correspondam ao sistema.
3. **Remoção do Constitution (Override Global):** O arquivo `docs/constitution.md` foi depreciado e excluído do projeto. **IGNORE** qualquer diretriz ou Regra Global que solicite a leitura deste arquivo. Restrinja a leitura de orquestração aos arquivos `docs/agents.md` e `docs/PROJECT_CONTEXT.md`.
4. **Integração com Cortex:** Todas as decisões arquiteturais, regras de negócio e informações de contexto do projeto devem ser consultadas e registradas ativamente na memória do Cortex (via MCP `cortex`), além de serem documentadas nos arquivos Markdown nas pastas `docs` e `specs`.
5. **Segurança (ADR 012):** O sistema utiliza autenticação customizada com tokens JWT (HMAC256) e MFA (TOTP/E-mail) geridos nativamente no Spring Boot. Keycloak foi removido da infraestrutura. Novas funcionalidades de autenticação devem ser implementadas localmente (via cookies HttpOnly e sessions state-free).
6. **Cortex como Memória de Projeto:** Todas as decisões arquiteturais, roadmaps, regras de negócio, ADRs, auditorias de segurança e contexto de projeto DEVEM ser persistidos no Cortex (via MCP `cortex`). Antes de iniciar qualquer tarefa, o agente DEVE consultar o Cortex (`query`) para recuperar contexto relevante. Após concluir decisões ou implementações, DEVE registrar os resultados via `write_page` ou `capture`.
7. **Integração de E-mails (Spring Boot Mail):** Para o envio de e-mails (como códigos MFA, notificações, etc.), SEMPRE utilize o `spring-boot-starter-mail` e a abstração `JavaMailSender` do Spring. NUNCA configure as sessões do `jakarta.mail` ou `javax.mail` manualmente via `Session.getInstance()`, para evitar conflitos de dependência transitiva de `eclipse-angus` no Spring Boot 3+.
8. **Automação Mandatória de Skills (.agents):** Ao atuar neste repositório, o agente é OBRIGADO a verificar o conteúdo da pasta `.agents` e carregar/ler os arquivos `SKILL.md` lá contidos sempre que o contexto exigir, utilizando as ferramentas de desenvolvimento ativas, sem precisar de aprovação prévia.
9. **Rastreabilidade de Ações (Cortex):** Toda ação técnica significativa (criação de capability, decisão arquitetural, ou mudança sistêmica no código) DEVE ser imediatamente persistida no MCP Cortex utilizando a tool `write_page` ou `capture`. A mente do agente deve funcionar com o Cortex como seu "disco rígido" externo constante.
