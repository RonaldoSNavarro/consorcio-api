# ADR 012 — Reversão para JWT Customizado e Retirada do Keycloak

**Data:** 2026-07-13
**Status:** Aprovada (decisão executada na sessão)

## Contexto e Motivação
A implementação da **ADR 008** (Migração para OAuth2/OIDC via Keycloak) foi concluída com sucesso técnico (Lotes 1, 2 e 3). No entanto, após análise da usabilidade (UX) e flexibilidade da interface padrão do Keycloak, optou-se por não prosseguir com a externalização da identidade utilizando essa ferramenta, conforme decisão do usuário.

## Decisão
1. **Revogar a ADR 008**: O Keycloak será removido da infraestrutura e não servirá mais como *Identity Provider (IdP)*.
2. **Retornar à Autenticação Customizada (ADR 001)**: O Backend (Spring Boot) volta a ser o responsável primário por emitir tokens JWT (`java-jwt`) e validá-os via `SecurityFilter`.
3. **Interface de Login no Frontend**: A responsabilidade de renderizar a interface de login retorna para o React (`LoginPage.jsx`), permitindo total controle sobre a estilização e UX.
4. **MFA**: A implementação de Autenticação em Dupla Etapa (MFA/TOTP), que era nativa do Keycloak, fica momentaneamente indisponível. Foi decidido que a reconstrução do MFA internamente no Spring Boot será avaliada para Lotes futuros, caso seja mandatório para o produto.

## Consequências
- Maior liberdade na construção visual do Frontend (não depende de temas de IdP externo).
- Remoção de complexidade de infraestrutura (Keycloak + PostgreSQL dedicado retirados do `docker-compose.yml`).
- Necessidade de implementação futura customizada para MFA, SSO, e controle avançado de sessão, se os requisitos mudarem.
